# Chapter 03 ID And Time Contract

IDs and timestamps are application contracts, not incidental database details.

## Rules

- Product IDs are UUIDv7 values generated by Rust application code.
- Job IDs and request IDs also use UUIDv7 unless a chapter states otherwise.
- New tables must not rely on database-generated IDs such as serial columns or
  `gen_random_uuid()` defaults.
- `created_at` and `updated_at` are set by the application clock boundary.
- Tests may inject fixed IDs and fixed timestamps.

## Why

UUIDv7 values are sortable enough for operational use and do not require a
central database sequence. Generating IDs in Rust also lets tests and domain
flows reason about IDs before persistence happens.

Keeping time behind a small clock boundary prevents flaky tests and makes it
clear which part of the system owns timestamp changes.
