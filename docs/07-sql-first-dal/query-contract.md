# Chapter 07 Query Contract

Move these query shapes from Rust-owned strings into named SQL files under
`db/queries/products/`. Use these exact file names and query names so mentor
reviews are consistent.

## Product Row Return Shape

Every query that returns a product row must return these fields:

```text
id
title
handle
description
price_cents
inventory_quantity
published
published_at
created_at
updated_at
```

## Query Files

### `db/queries/products/create_product.sql`

Query name:

```text
create_product
```

Parameters:

```text
id
title
handle
description
price_cents
inventory_quantity
published
published_at
created_at
updated_at
```

Returns:

```text
product row
```

Rules:

- `id` is generated by Rust as UUIDv7 before calling the query.
- `created_at` and `updated_at` are passed from the application clock.
- `published_at` is passed as a timestamp when `published = true`, otherwise
  `null`.

### `db/queries/products/list_products.sql`

Query name:

```text
list_products
```

Parameters:

```text
none
```

Returns:

```text
product rows
```

Ordering:

```text
order by created_at asc, id asc
```

### `db/queries/products/list_published_products.sql`

Query name:

```text
list_published_products
```

Parameters:

```text
none
```

Returns:

```text
product rows where published = true
```

Ordering:

```text
order by published_at desc nulls last, created_at asc, id asc
```

### `db/queries/products/update_product_publication.sql`

Query name:

```text
update_product_publication
```

Parameters:

```text
id
published
published_at
updated_at
```

Returns:

```text
product row
```

Rules:

- Set `published` to the input value.
- Set `published_at` to the input value.
- Set `updated_at` from the application clock.

## Done Check

Cornucopia-generated bindings must expose these query names, and the DAL must
wrap those generated calls instead of exposing generated query functions to
handlers.
