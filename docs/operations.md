# Operations Manual

This manual explains how to perform regular operational tasks on the Ahlan Commerce project. It goes beyond the basic commands and provides context on maintaining the system.

## Database Operations

### Creating New Migrations

1. Make changes to the `atlas.hcl` schema file or raw SQL definitions.
2. Generate a new migration using Atlas.
3. Review the generated SQL in `db/migrations`.
4. Apply using `make db-migrate`.

### Applying Production Migrations

Production uses Refinery from Rust, not Atlas. Keep `db/refinery_migrations/` aligned with reviewed schema changes, then run:

```sh
make db-migrate-prod
```

The API always runs embedded Refinery migrations during startup before creating its application database pool.

### Managing Raw SQL Queries

When new data access functions are needed:
1. Write the raw SQL query in a `.sql` file inside `db/queries/`.
2. Ensure you annotate the queries properly for Cornucopia.
3. Run `make cornucopia-generate` to generate the new Rust functions in `packages/db`.

## Reviewing Code Changes

When reviewing changes to the codebase, engineers should:
1. **Check CI**: Ensure all tests and lint checks pass.
2. **Check Documentation**: If an API route was added or changed, `make docs-api-check` must pass in CI, meaning the author ran `make docs-api` and checked in the updated OpenAPI or GraphQL schema.
3. **Verify API Contract**: The updated `openapi.json` and `schema.graphql` show the *exact* API surface that the pull request exposes. Review the changes to these files to understand the API contract drift.

## Troubleshooting

- **Database Connection Issues**: If `make run-api` fails with a Postgres connection refused error, verify that the Postgres service is running locally (`make db-start`) and that the credentials in `DATABASE_URL` match your local environment.
- **Missing Code generated by Cornucopia**: Ensure `make cornucopia-generate` is run whenever queries are modified. If it fails, double-check that the database schema is fully migrated (`make db-migrate`).
